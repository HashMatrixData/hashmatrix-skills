# Issue Report — 正文模板

> 按类型选用。中文撰写，写入临时文件后用 `gh issue create --body-file` 提交。
> 所有示例数据均为虚构占位，提交前替换为真实（脱敏）内容。

## Bug 模板

```markdown
## 症状
<一句话描述用户可见的问题>

## 复现步骤
1. <动词开头的步骤>
2. ...

## 实际结果
<实际发生了什么>

## 预期结果
<应该发生什么>

## 可复现性
<全环境必现 / 特定环境复现 / 特定数据复现 / 不稳定复现(频率: N/次) / 无法复现>

## 环境
- 环境: <dev / test / staging>　URL: <https://example.com/...>
- 严重程度: <P0/P1/P2/P3>
- TraceID: <如有>　首次出现: <时间>

## 错误日志 / 堆栈
\`\`\`
<完整日志，不截断；先脱敏掉 IP/账号/token>
\`\`\`

## 证据
<截图 / 录屏 / 响应体，按 SKILL.md Step 3 要求>
```

## Feature 模板

```markdown
## 用户故事
作为 <角色>，我希望 <功能>，以便 <价值>。

## 背景 / 动机
<为什么需要，解决什么问题>

## 验收标准
- [ ] <可验证条件 1>
- [ ] <可验证条件 2>

## 参考 / 设计
<参考实现、设计稿链接（脱敏）、约束>

## 优先级
<P1 阻塞当前迭代 / P2 重要不紧急 / P3 锦上添花>
```

## Task 模板

```markdown
## 任务描述
<要做什么，技术债/重构/文档/配置/运维>

## 背景
<为什么需要>

## 完成标准
- [ ] <可验证的完成条件 1>
- [ ] <可验证的完成条件 2>

## 优先级
<P1 / P2 / P3>
```
